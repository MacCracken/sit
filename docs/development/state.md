### Architecture call-outs (carried across v0.6.x + v0.7.x)

- **Patra handle caching**: `get_object_db()` (object_db.cyr) and `get_index_db()` (index.cyr) memoize per-process. `object_db_open()` / `index_db_open()` still exported for the wire-protocol callers that need an explicit fresh handle for the remote-side DB (every fetch/push targets a different file)
- **Wire transactions**: `copy_objects` (src/wire.cyr) wraps the insert loop in `patra_begin` / `patra_commit` since v0.6.5. `db_object_insert_raw` returns `1` for already-existed and `0` for actually-inserted (caller increments `copied` only on `== 0`)
- **Walk-reachable cache**: `walk_reachable_*` (src/wire.cyr) populates a `raw_cache` map of compressed bytes since v0.6.7; `copy_objects` reads from cache to skip the second source-DB read for commits + trees. **v0.7.3:** the cache is transport-independent (compressed bytes are bytes), so HTTP fetches benefit identically to local fetches
- **`obj_src` abstraction (v0.7.3)**: 16-byte tagged handle `{kind, payload}` wraps either a patra DB pointer (`OBJ_SRC_DB` = 0) or a `wire_http` handle (`OBJ_SRC_HTTP` = 1). `walk_reachable_tree` / `walk_reachable_from_commit` / `copy_objects` all take `obj_src` instead of a raw `src_db`; `obj_src_read_raw` / `obj_src_read_both` dispatch to either `db_object_*` (wire.cyr) or `http_remote_*` (wire_http.cyr). Lifetime: caller still owns the underlying handle and closes it explicitly (`patra_close` for DB, `wire_http_close` for HTTP)
- **`wire_transport_check` split (v0.7.3)**: `_readable` (file + http allowed) and `_writable` (still rejects http with `"push over http requires sit 0.7.5+"`); `cmd_clone` and `do_fetch` use `_readable`, `cmd_push` uses `_writable`. https + ssh stay gated everywhere with their original v0.7.6 / v0.7.8 pointers
- **HTTP body cap**: `WIRE_HTTP_MAX_BODY` = 16 MiB on both ends (server `SIT_SERVE_MAX_BODY`; client recv buffer growth). Matches `db_object_read_both`'s decompression ceiling so the failure modes line up regardless of transport
- **`POST /sit/v1/want`**: server endpoint live + advertised in capabilities (`"batch":true,"batch_max":512`); v0.7.5 wires the client side end-to-end via `walk_reachable_phased` (commit chain → tree-batch → blob-batch) + `obj_src_batch_prefetch` in `copy_objects`. Frame format at [ADR 0006](../adr/0006-batched-want-frame-format.md). Binding constraint on `SIT_WANT_MAX_COUNT = 512` is sandhi's `HSV_REQ_BUF_SIZE = 64 KiB` request buffer; client chunk size is 256 (`WIRE_HTTP_BATCH_CHUNK`). 10M-iteration fuzz harness (`fuzz_want_frame_decoder`) drives `_wire_http_decode_frames` against pseudo-random bytes — extracted from `http_remote_read_batch` specifically so the fuzz target doesn't need a TCP socket
- **Phased walker (v0.7.5)**: `walk_reachable_phased` + `walk_reachable_tree_batched` replace the sequential `walk_reachable_from_commit` + `walk_reachable_tree`. Phase 1 walks the commit chain sequentially (parent-of-parent has no parallelism). Phase 2 batch-prefetches every collected tree hex via `POST /sit/v1/want`. Phase 3 walks each tree from `raw_cache` — cache-aware via the new `_decompress_raw_into` helper that decompresses cached compressed bytes without re-fetching from the transport. Per-level sub-tree batching for nested directories. `obj_src_batch_prefetch` is no-op for OBJ_SRC_DB so file:// path is identical to v0.7.4 (`165 ms` median, no regression)
- **HTTP push + bearer auth (v0.7.6)**: `cmd_push` branches on URL scheme — `URL_SCHEME_HTTP` runs `_do_push_http` (capabilities probe, optional bearer-token load, FF preflight, walk reachable, POST each object via `http_remote_push_object`, POST ref via `http_remote_push_ref`, "everything up-to-date" short-circuit). Trust boundary: `_serve_rehash_and_insert` runs sigil's `hash_data` over every uploaded object's full `<type> <len>\0<content>` frame and refuses on hex mismatch. Bearer auth: `_serve_load_token` enforces 0600 perms + ≥16 chars + no control bytes; `_serve_auth_ok` does constant-time compare across `max(presented_len, token_len)` so timing doesn't leak prefix matches. Capabilities advertise `"auth":["bearer"]` when `--require-auth`, `"auth":["none"]` otherwise; `"push":true` always. Read endpoints (capabilities/refs/objects/want) stay anonymous in both modes
- **No-FFI thesis** (ADR 0001 + ADR 0007): no libssl, no libcrypto, no fdlopen-bridged dlopen. HTTPS via libssl will not ship; first-party Cyrius TLS is the only path that unlocks `https://` later. SSH (v0.7.8 — bumped from v0.7.7 when owl's library-export request slotted in front of it) is the canonical encrypted-over-internet transport (sit consumes the SSH binary as a process boundary, not an FFI dep — same separation as git's ssh:// support). HTTP is loopback / private-network / behind-tunnel; bearer auth is local-process-snoop defence, not a TLS substitute
- **Buffered stdout**: 206 sites in src/*.cyr route through `stdout_write(data, len)` in src/util.cyr since v0.6.8; flushed in main.cyr trailer before `SYS_EXIT`. STDERR (`eprintln`) stays direct
- **exec_vec envp**: empty (cyrius stdlib `lib/process.cyr` passes a NULL-only envp; no env-var inheritance in `cmd_owl_file`). Any future curated-envp shape would be a deliberate widening
- **patra BATCH mode (NOT enabled)**: investigated in v0.6.10 and reverted — durability regression with no perf gain on sit's bench shape (cached handle never closes, so pending writes would sit in the kernel writeback window). Reasoning at the `get_object_db` / `get_index_db` call sites; revisit when sit grows explicit `patra_flush()` at command exit

## Storage layout (sit repos on disk)

- **`.sit/HEAD`** — symbolic ref (`ref: refs/heads/main\n`)
- **`.sit/refs/heads/<name>`** — branch ref, 64-char hex + `\n`
- **`.sit/refs/tags/<name>`** — tag ref, same format
- **`.sit/refs/remotes/<remote>/<branch>`** — remote-tracking ref
- **`.sit/objects.patra`** — patra DB, schema `objects(hash STR, ty INT, content BYTES)`
- **`.sit/index.patra`** — staging index, schema `entries(path STR, hash_hex STR)`
- **`.sit/config`** — local config (`user.name`, `user.email`, `remote.<name>.url`)
- **`~/.sitconfig`** — global config (same format)
- **`~/.sit/signing_key`** (0600) / `~/.sit/signing_key.pub` (0644) — ed25519 seed + pubkey hex
- **`.sit/logs/HEAD`**, **`.sit/logs/refs/heads/<branch>`**, **`.sit/logs/refs/remotes/<remote>/<branch>`** — reflog (v1.1.0): append-only git-format journal, one line per ref movement (`<old64> <new64> <name> <email> <ts> +0000\t<msg>`); see [ADR 0010](../adr/0010-reflog-and-recovery.md). Tags are not logged
- **`.sit/MERGE_HEAD`** — in-progress merge marker (cleared on commit or `--abort`)
- **`.sit/shallow`** — shallow-clone boundary markers (v0.8.12): one 64-char hex per line, each a present commit whose parents were not fetched. Written by `sit clone --depth <n>`; read by `sit log` to stop cleanly at the boundary. Absent for full clones
- **Legacy**: `.sit/objects/<xx>/<yy...>` loose files and plaintext `.sit/index` auto-migrate on first access — see [arch 002](../architecture/002-loose-objects-until-patra-bytes.md)

## Consumers / integration

- **Dist bundles (two profiles):** `dist/sit.cyr` (full, `[lib]` → `cyrius distlib`) is the default for consumers needing the write/sync surface; **`dist/sit-read.cyr`** (lean read-only, `[lib.read]` → `cyrius distlib read`, **v1.3.0**) is for report-only consumers — it drops `sign` + the network stack so it compiles into a consumer with no shim constants and no wire warnings. Both are tracked in-repo and regenerated together when a shared read-path module changes (CLAUDE.md Work-Loop step 8).
- **thoth** (pre-1.0, downstream) — its status bar / tool-call diffs consume the **read profile** (`dist/sit-read.cyr`) for branch / status / diff via the `sit_repo_*` API (0.13.0 git producer). Vendors sankoch as its own dep for the git-object inflate path.
- **owl** (pre-1.0, downstream) — consumes sit for git-marker gutter decorations once both ship. sit's `owl-file` command falls back to raw content when owl isn't on PATH. A candidate to move onto the read profile alongside thoth.

## Known footguns (tracked)

See [roadmap.md § Longer horizon](roadmap.md#longer-horizon) for the full list. Highlights:

- ~~**Push to checked-out branch**~~ — **closed in v0.8.4.** All three transports (file://, http://, ssh://) refuse pushes whose target ref is the remote's checked-out HEAD; initial pushes to empty remotes still succeed.
- ~~**`sit fsck` reachability**~~ — **closed in v0.8.5.** Output gains `dangling <type> <hex>` + a third counter; walker covers refs/heads/tags/remotes + detached HEAD + staging index, follows all commit parents.
- ~~**Wire walker single-parent capture**~~ — **closed in v0.8.7.** `parse_commit_body` now exposes every parent via a vec at `out+48`; `walk_reachable_phased` / `is_ancestor` / `is_ancestor_in_db` follow all edges, so clone / fetch / push no longer drop merge subgraphs and the FF gate doesn't misjudge ancestry across merges.
