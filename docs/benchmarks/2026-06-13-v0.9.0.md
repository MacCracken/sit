# Benchmark snapshot — 2026-06-13 (v0.9.0 closeout)

The pre-v1.0.0 baseline. Confirms the v0.8.x feature work + the v0.9.0 closeout
(bug fixes, dead-code removal, the `commit_parents_of` refactor) left
performance **flat** vs the [v0.8.12 baseline](2026-06-13-v0.8.12.md). Full
per-group decomposition lives there; this records the headline numbers and the
deltas only.

**Host**: Linux x86_64, cyrius 6.2.2, sit 0.9.0. Single-run averages.

| Bench | v0.8.12 | v0.9.0 | Δ |
|-------|--------:|-------:|---|
| `sha256-65536B` | 195 µs | 202 µs | flat (variance) |
| `copy-objects-100` | 132 µs | 138 µs | flat |
| `ed25519-verify` | 6.907 ms | 7.129 ms | flat |
| `lcs-diff-1000x1000` | 40.27 ms | 41.44 ms | flat |
| `lcs-diff-4000x4000` | 593 ms | 607 ms | flat |
| `is_ignored-200pat` | 143 µs | 141 µs | flat |
| `blob-hash-1048576B` | 5.189 ms | 5.309 ms | flat |

All movement is within run-to-run variance — no regression from the closeout
changes. The `commit_parents_of` extraction touches `is_ancestor` /
`find_merge_base`, neither of which is on a benched hot path, and the graph /
prune guards are branch-only.

## v0.6.x scoreboard (carried, still the cumulative perf story)

The v0.6.0→0.6.12 wins remain the headline: `add-1MB` −48%, `add-64KB` −43%,
`clone` −30%, `log` −17%, `status` −9% from the v0.6.0 baseline. No v0.8.x /
v0.9.0 release moved that scoreboard — the line has been feature- and
correctness-driven, not perf-driven, since v0.7.x. SHA-256 (~200 MB/s via
sigil's SHA-NI path) and `zlib_compress` remain the `sit add` floors.
