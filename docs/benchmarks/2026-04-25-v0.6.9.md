# Benchmark snapshot — 2026-04-25 (v0.6.9) — sit-side v0.6.x perf arc closed

Captured after P-06 (smarter decompression sizing) + P-15 (LCS DP table to `fl_alloc`). Pairs with [`2026-04-25-v0.6.8.md`](2026-04-25-v0.6.8.md). This is the last v0.6.x sit-side perf release; further headline movement requires upstream work.

**Host**: Linux 6.18.22-1-lts x86_64, cyrius 5.6.35, sankoch 2.0.3, sit 0.6.9, git 2.53.0
**Bench config**: `RUNS_LIGHT=20 RUNS_HEAVY=10`

## v0.6.9 — what changed

Two small hygiene/edge-case items, neither expected to move the synthetic bench (and neither did):

- **P-06 — smarter decompression sizing.** Three call sites updated (`src/object_db.cyr:read_object`, the loose-migration path, `src/wire.cyr:db_object_read_both`). Initial multiplier dropped from 16× to 4× — most sit objects (commits, trees, source-shape blobs) decompress at ratio ~2-3× and fit easily; legitimately high-ratio outliers retry at the 16 MiB ceiling. **Critically, retry only on confirmed `-ERR_BUFFER_TOO_SMALL` (-2)** — other negative codes mean the stream is genuinely corrupt and more memory won't help. Fail fast on real corruption; cuts 75% of decompression-buffer memory in the common case.
- **P-15 — LCS DP table to `fl_alloc`.** `lcs_diff` in `src/diff.cyr` now allocates the DP table via `fl_alloc` (mmap-backed for large allocs) and `fl_free`s before returning. Previously the table sat on the bump heap permanently — up to 128 MB of process RSS held for the life of the command even though the table is only used during the LCS computation. Pure memory hygiene; no wall-clock change.

Why nothing moved on the synthetic bench:
- **P-06**: the 4096-byte floor on `dec_cap` dominates for small objects (the bench fixture's blobs are ~17 bytes, trees ~4KB, commits ~300B — `blen * 4` is well under 4096 for all of them so the floor wins). Real win shows up on objects with `blen > 1024` — i.e., source files in actual repos. For a 1 MiB compressed blob, dec_cap goes 16 MiB → 4 MiB. **75% memory savings**, no wall-clock change since both fits work the same.
- **P-15**: bench's `diff-edit` LCS is a 500-line × 500-line DP table = 2 MB. Bump-heap or mmap-heap, the alloc cost is similar. The win is RSS-cleanliness post-diff (the table goes back to the kernel via `munmap`-on-`fl_free` instead of squatting on the bump heap until process exit). Visible in `/proc/<pid>/status` mid-process; not a wall-clock signal.

## What sit-side perf can move next — and what it can't

**Sit-side has plateaued.** Cumulative `log -18%`, `clone -30%` from a half-dozen small targeted changes. The remaining wall-clock budget on every bench row is dominated by costs the sit codebase doesn't own:

| Workload | What dominates | Where the next win lives |
|---|---|---|
| `add` (any size) | sankoch zlib_compress + sigil SHA-256 | **sankoch** roadmap (DEFLATE throughput); **sigil** roadmap (SHA-256 throughput) |
| `status` | sigil SHA-256 over file content × N files | **sigil** roadmap |
| `clone` | per-object insert + WAL fsync | **patra** 1.7.0 `INSERT OR IGNORE` (drops inner has-check); **patra** 1.8.0 WAL group commit (auto-batched fsync, would replace sit's manual `patra_begin`/`patra_commit` in `copy_objects` with implicit batching everywhere) |
| `diff` | sankoch zlib_decompress + LCS algorithm | **sankoch** roadmap; algorithmic LCS (Myers' O((N+M)D), audit P-14 — currently only deferred) |

All three dep asks are filed on the respective lib roadmaps as of v0.6.4. When any of them ships, sit can drop the corresponding workaround / pick up the matching improvement without further sit-side code change.

## What ships in v0.6.9 and where to next

This release closes the sit-side v0.6.x perf arc. Roadmap moves:
- All P-NN items in scope for the perf arc are now either shipped (P-01, P-02, P-03, P-04, P-05, P-06, P-08, P-10, P-12, P-15, P-17, P-18, P-25 — and S-24 which folded into P-01) or moved out of scope to the v0.7.x or v0.8.0+ work plan (P-07 bump-arena reset, P-11 sit add upsert via patra UPSERT, P-13 glob bucket, P-14 Myers diff, P-16 fsck --fast, and the micro-wins P-19/P-21/P-23).
- Next sit ship target: **v0.7.0 — network transport (HTTP/SSH)**, the major roadmap item that's been queued since v0.5.0 shipped local-path transport.

Or wait on a patra/sigil/sankoch ship and revisit perf with a fresh upstream baseline — sit-side `clone`, `status`, `add`, `diff` are all dep-bound at this point and pure sit-side work would only be polish.
