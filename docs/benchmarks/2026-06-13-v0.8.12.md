# Benchmark snapshot — 2026-06-13 (v0.8.12)

First snapshot since the v0.6.x perf arc. Re-establishes `cyrius bench` as a
per-release gate (the harness had drifted — `bench_copy_objects_per_row` was
calling the pre-v0.7.3 3-arg `copy_objects`, so its number was noise; fixed
here). Adds the three Phase-3 targets scoped during the 0.6.0 audit but never
landed: LCS diff, the `.sitignore` matcher, and blob hashing.

**Host**: Linux x86_64, cyrius 6.2.2, sit 0.8.12. Single-run averages (not
multi-sample medians) — treat as order-of-magnitude, not regression-grade to
the nanosecond.

## Phase 3 — new in v0.8.12

### LCS diff (`compute_file_diff` — drives `sit diff`/`show`)

| Size | Time/op | Notes |
|------|--------:|-------|
| 100 × 100   | 438 µs | every 10th line edited |
| 1,000 × 1,000 | 40.27 ms | DP table ~1M cells |
| 4,000 × 4,000 | 593.46 ms | ~16M cells; near the `n1/n2 > 8192` cap (S-07) |

Quadratic in line count, as expected — the DP fill dominates. The `fl_alloc`/
`fl_free` DP table (v0.6.9) keeps the 4000² case from squatting 128 MB on the
bump heap.

### `.sitignore` matcher (`is_ignored` — every `sit add`/`status` walk entry)

| Patterns | Time/op | Per-pattern |
|---------:|--------:|------------:|
| 10  | 7 µs   | ~0.7 µs |
| 50  | 36 µs  | ~0.72 µs |
| 200 | 143 µs | ~0.72 µs |

Linear in pattern count (last-match-wins scans all patterns). ~0.72 µs/pattern
against a 6-segment path — the non-anchored any-level fallback walks each `/`
boundary, so cost scales with path depth too.

### Blob hashing (`hash_blob_of_content` — per-file cost of `sit add`)

| Content | Time/op | Implied throughput |
|--------:|--------:|-------------------:|
| 1 KB    | 6 µs    | ~171 MB/s |
| 64 KB   | 327 µs  | ~200 MB/s |
| 1 MB    | 5.189 ms| ~202 MB/s |

Throughput tracks sigil's SHA-256 ceiling (the framing memcpy is negligible).
This is the floor `sit add` pays per file after the working-tree read.
